# Frequency Dictionary Tree App

**Desktop-приложение на C# (.NET 9) для работы с частотным словарём на основе бинарного дерева**

Приложение реализует полный цикл обработки текста: токенизацию со стеммингом, построение частотного словаря, хранение данных в собственной реализации бинарного дерева и сортировку выбором. Проект демонстрирует глубокое понимание структур данных, алгоритмов и современных практик разработки .NET.

---

## Возможности

- **Построение частотного словаря** — автоматический подсчёт частоты слов из произвольного текста
- **Бинарное дерево** — эффективное хранение и поиск пар «слово → частота» с поддержкой упорядоченного обхода
- **Стемминг и токенизация** — нормализация слов к основе (stemming) перед подсчётом частоты (учёт морфологии языка)
- **Сортировка выбором (Selection Sort)** — демонстрация классического алгоритма сортировки применительно к словарю
- **Современный десктопный интерфейс** — удобный GUI для работы с текстом
- **Полная объектно-ориентированная архитектура** — разделение на модели, сервисы и слой представления

## Установка и запуск

### Предварительные требования
- [.NET 9 SDK](https://dotnet.microsoft.com/download/dotnet/9.0) (или новее)

### Пошагово

```bash
# 1. Клонирование
git clone https://github.com/AxineBro/frequency-dictionary-tree-app.git
cd frequency-dictionary-tree-app

# 2. Переход в папку проекта
cd FrequencyDictionaryTreeApp

# 3. Восстановление зависимостей
dotnet restore

# 4. Сборка
dotnet build

# 5. Запуск
dotnet run
```

**Альтернатива**: откройте файл `FrequencyDictionaryTreeApp.sln` в Visual Studio 2022+ и нажмите **F5**.

## Архитектура и дизайн

Проект построен как демонстрация **чистой, расширяемой и поддерживаемой** архитектуры в десктопном .NET-приложении. Основные принципы и паттерны:

### Применённые паттерны проектирования

- **Facade**  
  Классы в слое `Services` выступают в роли фасада, скрывая сложность работы с деревом, стеммером, токенизацией и сортировкой. UI-слой общается только с этим фасадом, не зная деталей реализации.

- **Factory**
  Создание узлов дерева, токенов или стеммеров вынесено в фабричные методы / классы, что позволяет легко подменять реализацию (например, разные стеммеры для разных языков).

- **Single Responsibility Principle (SRP)**  
  Каждый класс отвечает ровно за одну вещь:  
  • `BinaryTree<T>` — только операции с деревом  
  • `Stemmer` — только приведение слов к основе  
  • `TextTokenizer` — разбиение текста на токены  
  • Сервисы — оркестрация процесса

- **Open-Closed Principle (OCP)**  
  Логика открыта для расширения, но закрыта для модификации:  
  • Можно добавить новый алгоритм сортировки, не трогая существующий код  
  • Можно подключить другой стеммер (Porter, Snowball, собственный)  
  • Дерево можно заменить на AVL / красно-чёрное без изменения потребителей

- **Слоистая архитектура** (Layered / Clean Architecture light)  
  ```
  Presentation (WinForms / WPF)  
  ↓  
  Application Services  
  ↓  
  Core (модели, BinaryTree, алгоритмы, Stemmer)  
  ```
